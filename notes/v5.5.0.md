
### Added
- **License → entitlements**: the license's plan now actually gates the
  product — a plan catalog (workforce / endpoint / enterprise / pilot /
  partner / internal) maps to enabled modules and allowed update channels;
  navigation, the module API, and backend enforcement all honor it, and the
  update channel is clamped to the licensed tier (customers can never resolve
  internal `edge` builds). Explicit per-license entitlement overrides supported.
- **Forced rotation of admin-issued passwords**: accounts created by an admin
  and admin password resets require the owner to set their own password on
  next login — the session is limited to the change-password flow until then,
  and the login screen walks the user through it. LDAP accounts excluded
  (their password lives in AD).
- **Agent Deployment works end-to-end**: the deploy dialog's install commands
  now match the shipped artifacts; installer downloads authenticate with the
  enrollment token itself (no session needed on the new machine, audited,
  non-consuming); first-agent bootstrap token can be issued from the UI.
- One-command installer published at a stable public URL and kept in sync by
  the release pipeline; customer image access stays private with per-customer
  deploy tokens.
- Updates page: the "Last update action" banner can be dismissed, and stale
  outcomes for versions the server has since reached clear automatically.

### Fixed
- EDR alert-storm dedup refined: the unconditional dedup is process-scoped
  (identical repeats collapse; distinct processes never merge) while the
  admin-configured suppression window keeps its rule-wide scope; long process
  paths no longer risk oversized keys. WebsitePolicy block-hits now emit the
  website_block telemetry event (previously only UAM-policy blocks did).
- SQLite installs (demo/dev) now receive schema migrations at startup —
  schema drift no longer breaks endpoints after upgrades; the runtime demo DB
  is no longer tracked in git.
- Screenshot/evidence privacy and access controls hardened.
- Rebrand compatibility audit: six more legacy-name detection sites restored
  (pre-rebrand installs are fully recognized again, including agent service
  accounts in UAM identity classification).

### Security
- Password hashing moved off the unmaintained passlib to bcrypt 5 with
  explicit 72-byte handling; existing hashes keep verifying.
- Bandit/pip-audit security gate is clean: real fixes (non-crypto hash
  annotations) plus per-line justified suppressions; test dirs excluded.
