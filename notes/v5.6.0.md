## [5.6.0] — 2026-07-13

### Added
- **Antivirus trust chain (Phases 1–2C)**: detection bundles are now signed
  with Ed25519 — agents verify a canonical envelope against a public key ring
  delivered via config push, with key rotation/revocation, a durable
  anti-rollback version watermark, and last-good protection on any failure.
  YARA rules ride the same signed chain (digest-bound to the envelope; raw
  rule text is never compiled). Signature DB freshness and per-endpoint
  applied-version visibility land in the console, a default YARA baseline
  ships in the signed chain, and policy enforcement gains an effective-policy
  pinning model with an exclusions + approvals workflow.
- **UAM behavioral rules foundation**: behavioral evaluation with a
  condition builder, read-only preview and dry-run, durable shadow
  aggregation for rule tuning, and policy targeting (users / hostnames /
  departments) with validated pickers.
- **Alerts & incidents maturity**: human-readable incident numbers
  (backfilled), role-based alert privacy projection (path/URL/IP details are
  masked down to what the viewer's role warrants), and EDR operational
  context on alerts.
- **Agent trust recovery**: a stranded agent (tamper flag, failed secret
  rotation) can be recovered with an admin-issued, short-lived, single-use,
  device-bound token — fully audited, rate-limited, and completed only after
  the endpoint attests possession of the new secret. No re-enrollment needed.
- **Dashboard overview rebuilt on real data** with management-grade
  semantics, plus a sidebar redesign (fixed dark rail, cleaner collapse).

### Fixed
- Screenshot evidence lists no longer hide evidence older than 24h; evidence
  viewing/downloading is RBAC-gated, audited, and never exposes storage
  paths.
- First-Seen-Binary false positives: trusted-context tuning (path category,
  signer, user-writability) scopes down alert noise on known-good binaries.
- UAM users list and user-profile loads are significantly faster on real
  data.
- Audit chain: pre-upgrade (v1) history forests are preserved and classified
  under a serialized v2 checkpoint instead of failing validation.
- Sidebar collapse geometry regressions fixed and the fixed-dark skin unified
  with the base layout (the collapse control no longer overlaps content).

### Security
- **Database security governance (ITGC/DBSEC)**: audit log entries are
  hash-chained with fail-closed consistency proof (with a dedicated
  PostgreSQL proof suite), schema migration governance controls detect
  out-of-band changes, and runtime database roles are inspected for
  least-privilege + TLS posture.
- Legacy HMAC bundle-signing material removed server-side and cleared from
  endpoints (no downgrade path back from Ed25519); the unsigned YARA compile
  path was deleted from the agent.
- Unauthenticated trust-recovery rejections can no longer flood the audit
  chain (per-IP audit cap; rejections still logged).

### Changed
- Dependencies: uvicorn 0.51, croniter ≥6.2.4, vite 8.1.4, ruff 0.15.21,
  GitHub Actions checkout@v7 / setup-python@v6 / setup-node@v6.
  (TypeScript 7 held back for a dedicated migration.)

### Known limitations
- Per-append audit-chain validation cost grows with history length (#398);
  scheduled for optimization before first customer deployment. Antivirus
  remains labeled preview-grade pending the enterprise-readiness audit items.
