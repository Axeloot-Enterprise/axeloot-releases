
### Added
- **Automatic update discovery (zero configuration)**: every install polls the
  public Axeloot release feed daily and surfaces new releases on the Updates
  page with release notes — no repos, tokens, or settings for the customer.
  Air-gapped installs fall back to uploaded bundles automatically; hardened
  sites can disable outbound checks with `AXELOOT_UPDATE_SOURCE=local|none`.
- Release pipeline publishes the public feed (manifest + notes) on every tag.

### Fixed
- Login screen: real version badge (was hardcoded v5.0); app route no longer
  leaks to the address bar pre-auth and deep links restore after sign-in;
  brand block is no longer a dead link.
- Updates page crash from the removed settings fetch; "Web application" vs
  "Endpoint agents" sections clearly separated.
- Update-check caching honors settings changes immediately; error results
  expire in minutes, not hours.
- Agent dependency manifests reconciled (psutil/Pillow caps vs floors,
  pywebview 6 in both files).

### Changed
- Pre-release 8-angle review landed 14 fixes incl. removal of the dead
  `server/auth.py` and faster air-gap update checks.
- Dependencies: pillow 12, psutil 7, pytest 9 + pytest-asyncio 1.4, React 19,
  vite 8, TypeScript 6, node 26 builder; bcrypt 5 deliberately rejected
  (breaks passlib) pending an auth-hashing migration.

### Fixed
- **Screenshot command flow produced no evidence** (risk: medium): three root
  causes fixed — the server never pushed the `allow_screenshot_commands` gate
  (every `request_screenshot`/`start_session_recording` was refused as
  `disabled_in_lightweight_phase`), a successful user-session helper capture was
  never uploaded (no `Screenshot` row was created), and
  `start_session_recording` raised `AttributeError` because the Phase 18 module
  split stranded the recording methods as dead code on `SystemInfo`.
  `/api/agents/config` now derives `allow_screenshot_commands` from the
  effective MonitoringPrivacy policy (deny by default, admin-created and
  audited, forced off by privacy mode); the agent uploads captures to
  `/api/agents/screenshot` with a path-containment check and deletes the local
  copy; session recording is restored on the Session-0-safe IPC path.
  - Migration: screenshots stay disabled until an admin creates a
    privacy-controls entry with `screenshot_enabled: true` for the target scope.
  - Rollback: delete the privacy-controls entry (agents fall back to deny).

### Security
- **Server-side URL privacy enforcement for browser telemetry** (risk: medium):
  query strings and fragments are now stripped server-side before any
  persistence — activity ingestion (`TelemetryEvent.data.url`), the
  block-hit path (previously stored the raw reported URL), alert provenance
  `triggered_url`, and the heartbeat browser-extension health `last_url`.
  - Affected: deployments receiving telemetry from pre-PR#202 or third-party
    agent builds (current agents already sanitize client-side).
  - Mitigation: defense-in-depth — the server no longer trusts the client to
    sanitize; command results never include the server's OCR preview.
  - Rollback: revert the commit; no schema change involved.
