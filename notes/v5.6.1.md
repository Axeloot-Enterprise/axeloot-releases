
Patch release: the weekly release-train review gate ran against the v5.6.0
window after v5.6.0 had already been cut; this release carries its confirmed
correctness/security fixes. No new features.

### Upgrade notes
- **PostgreSQL deployments with existing audit history must initialize the
  Audit Chain v2 checkpoint when upgrading to v5.6.x** — run
  `python server/audit_chain_recovery.py dry-run` then `apply` (procedure:
  `docs/security/AUDIT_CHAIN_V2_RECOVERY.md`; steps now folded into
  UPGRADE.md §3.5b). Without it, audited actions — **including login** — fail
  closed after upgrade. Startup now logs a CRITICAL notice with the exact
  command when this state is detected. Fresh installs and SQLite demo boxes
  initialize automatically. (v5.6.0 shipped the fail-closed behavior without
  these notes — if you already upgraded and are locked out, run the recovery
  command above.)

### Security
- Agent: `website_policy_signing_key` received at service start is now stored
  via the DPAPI-backed secret store instead of plaintext config (the heartbeat
  refresh path was already correct).
- Trust recovery: the consume endpoint's brute-force rate limit now runs
  before validation; previously only successful consumes were ever counted,
  so failed guessing attempts were never throttled.
- Enrollment and response-engine audit writes are best-effort again: an
  audit-chain failure no longer replaces an enrollment rejection with a 500
  or aborts a response playbook mid-run.

### Fixed
- SLA compliance report no longer counts late-resolved incidents as "met";
  the incident-response summary counts resolved-late breaches.
- AV bundle rollback preserves the target version's real YARA rule count in
  the signed envelope (was hardcoded 0).
- Expired threat indicators now trigger a bundle republish, so they actually
  drop out of the served signed bundle.
- Incident number assignment retries under a savepoint on concurrent-creation
  collisions instead of returning a 500.
- Dashboard Security Status footer no longer renders unavailable metrics as
  literal zeros.
- Responder role regained its case-editing controls in the cases workspace
  (a stale admin/analyst-only global was hiding them; server-side RBAC always
  permitted responders).
- UAM rule wizard no longer silently discards a newly added condition for a
  field whose stored value was in a legacy shape.
